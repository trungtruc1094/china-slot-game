# Decision Log: China Slot Game PRD

## Decisions

- 2026-06-01: Fast-path PRD creation approved by user after BMAD help routing.
- 2026-06-01: PRD scope centers on separating browser client from backend authority so operational metrics can be configured, measured, and audited.
- 2026-06-01: Backend is required to own RNG, bet validation, payout calculation, balances, session state, game configuration, and audit records.
- 2026-06-01: Client is required to become presentation and animation only for reward-bearing play.
- 2026-06-01: Operational controls must tune published game configuration and budget limits, not manipulate individual user outcomes.
- 2026-06-21: Database persistence is required before Tevi Mini App integration because top-up, webhook, retry, wallet credit, audit, and restart-recovery flows need durable PostgreSQL-backed state and idempotency.
- 2026-06-21: Persistence scope preserves the current non-cash reward boundary and explicitly excludes Tevi identity, Tevi top-up implementation, cash-out, redemption, and game math redesign.

## Deferred

- Legal/compliance review requirements are marked as phase-blocking before any real-money, cash-equivalent, crypto, or redeemable reward launch.
- Exact target RTP, hit rate, volatility, and budget amounts remain open until the host confirms business constraints.
- Exact database schema, migration tooling, locking strategy, retention windows, and deployment migration mechanism are deferred to architecture while the PRD addendum defines the required product behavior.
